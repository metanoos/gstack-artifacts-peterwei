# QA Test Plan: Desktop GLM Task Trace Proof

Generated by `/plan-eng-review` on 2026-07-17. This is the primary `/qa` and `/qa-only` input for the executable desktop slice.

## Objective

Prove that one GLM Continue interaction creates exactly two independently durable artifact commits at the correct moments:

1. `Step & Continue` commits the AUTHOR Step, exact prepared request, dispatch intent, and outbox before provider I/O.
2. `Accept` commits a separate MODEL Step only after a complete successful result passes native validation.

Failure, rejection, stale review, local Stop, and unknown execution preserve the AUTHOR Step exactly once and create no MODEL Step.

## Test Layers

| Layer | Purpose | Runtime |
| --- | --- | --- |
| pure reducer | exhaustive legal/illegal transition table and effect intent determinism | Node 24 + `tsx --test` |
| shared conformance | exact schema/canonical/event vectors | TypeScript package + client + MCP |
| native integration | SQLite transactions, signing boundary, recovery, secrets, stream journal | Rust `cargo test --locked` |
| relay verification | independent event/signature acceptance and exact-byte publication | Go tests + `npm run verify:relay` |
| renderer | prompt inspector, blocked states, preview/review, attribution, replay | client tests/browser fakes |
| packaged app | real webview/Tauri IPC/database/restart wiring | test-only macOS WebdriverIO service |

## Mandatory Recovery Diagram

```text
Draft AUTHOR turn
├─ dirty/missing context -> blocked; no signature/db/provider I/O
├─ candidate validation fails -> no db/provider I/O
├─ dependency changes -> candidate invalidated
└─ confirm Step & Continue
   ├─ SQLite tx/sign fails -> no AUTHOR Step, no intent, no provider I/O
   └─ tx commits AUTHOR Step + prepared op + dispatch intent + outbox
      ├─ crash before send -> durable not-sent; safe resume
      ├─ send ambiguous -> UNKNOWN; no automatic retry
      ├─ terminal failure/local Stop -> AUTHOR remains; no MODEL
      └─ success -> pending review
         ├─ reject -> AUTHOR remains; no MODEL
         ├─ stale target -> preserve result; no MODEL
         └─ accept
            ├─ native validation/sign fails -> pending review
            ├─ accept tx fails -> pending review
            └─ tx commits MODEL Step + head + outbox
               ├─ projection crash -> restart applies exactly once
               ├─ append ArtifactIntent -> same trace
               └─ reply ArtifactIntent -> new trace + exact reply edge
```

## Deterministic Failpoint Suite

Use a temporary real `press.sqlite3`, fixed clock, fixed signer, fake provider transport, named failpoints, and the same database reopened after each failure.

Required failpoints:

- before candidate persistence;
- after candidate persistence but before confirmation;
- before AUTHOR signature;
- after signature but before SQLite begin;
- after each AUTHOR transaction write and before commit;
- after AUTHOR commit but before effect execution;
- before provider connect;
- after remote connect but before locally observed acknowledgement;
- after each coalesced stream journal write;
- before terminal response collapse;
- after response collapse but before pending-review projection;
- before MODEL native validation/signature;
- after MODEL signature but before accept transaction;
- after each MODEL transaction write and before commit;
- after MODEL commit but before editor projection;
- after outbox enqueue but before relay send/acknowledgement.

For every failpoint assert:

- exact event count and IDs;
- exact current head;
- operation execution/application state;
- pending effect intents;
- provider request count;
- MODEL application count;
- outbox destinations and referenced event ID;
- restart recovery state and available user remediation.

## Reducer Transition Coverage

- Every legal transition produces a deterministic next state and idempotent effect-intent set.
- Every illegal transition is rejected without persistence or effects.
- Duplicate input events and duplicate IPC delivery are idempotent.
- `unknown` never collapses to failed/cancelled without evidence.
- `cancelled-before-dispatch` requires proof no send occurred.
- `stop-requested -> stream-stopped-locally` becomes `unknown` absent provider acknowledgement.
- Abandoned UNKNOWN attempts remain ineligible if a late result arrives.
- A linked retry records duplicate-output/cost acknowledgement and never rewrites its predecessor.
- Execution and application states remain orthogonal.

## Independent Conformance Corpus

Expected outputs are checked-in constants and are never generated by the implementation under test.

Corpus coverage:

- RFC 8785 official examples and invalid I-JSON cases;
- fixed Nostr secret/public keys, timestamps, kinds, tags, content, IDs, and signatures;
- Unicode key order, non-BMP characters, escape sequences, and UTF-8/UTF-16 boundary cases;
- invalid NaN/Infinity/lone-surrogate inputs where representable at the API boundary;
- valid and invalid task turn partitions;
- valid and invalid presentation Run partitions;
- AUTHOR/MODEL/EXTERNAL/FILESYSTEM origins and stable voice IDs;
- valid and invalid context tag/turn agreement;
- append and reply ArtifactIntent hashes;
- unknown relationship roles preserved but not indexed as known semantics;
- publication rejection for local TaskOperation records.

Run the same fixtures through:

1. `packages/protocol`;
2. client schema/event integration;
3. MCP schema/event integration;
4. Rust native validation/signing; and
5. independent Go relay verification.

Agreement means exact canonical bytes, hashes, event IDs, and signatures—not merely equivalent parsed objects.

## Native Signing Boundary

- No registered Tauri command accepts an arbitrary Nostr event/template for signing.
- `prepareAuthorTurn` independently validates current head, ActionContract, typed turns/Runs, context references, ArtifactIntent, and size limits.
- `commitAuthorAndDispatch` loads only the frozen candidate from SQLite, revalidates it, signs it, and commits it.
- `acceptModel` loads only a durable succeeded result and hashed ArtifactIntent, validates them, and commits the MODEL Step.
- Mutating any webview-supplied display/summary field cannot change signed bytes.
- Unknown action, stale candidate, mismatched expected head, invalid Run partition, and oversized body fail closed.

## Exact Request and Context Tests

- Prompt Inspector bytes equal the persisted and dispatched body exactly.
- Provider/model/preset/action/lens/rule/context/target-head changes invalidate approval.
- Dirty context identifies exact files and performs no automatic Step.
- Manually Step the named file, rerun preflight, and observe the new exact context Step ID.
- Context references appear only on the resulting MODEL Step and disappear from the next Step unless reused.
- Context backlinks use the dedicated context tag and never appear in citation/quote/Coin/reply totals.
- Empty allowed context, maximum context, multibyte byte-boundary, and over-budget context behave explicitly.

## Two-Step Artifact Tests

- Success creates AUTHOR Step first and MODEL Step only on Accept.
- Reject preserves AUTHOR Step and creates no MODEL Step.
- Provider HTTP/network failure preserves AUTHOR Step and creates no MODEL Step.
- Local Stop preserves AUTHOR Step, labels outcome unknown, and creates no MODEL Step.
- Ambiguous crash preserves AUTHOR Step, labels outcome unknown, and creates no MODEL Step.
- Stale target preserves AUTHOR Step/result evidence and creates no MODEL Step.
- Double Accept, duplicate response, and duplicate IPC delivery create at most one MODEL Step.
- Append intent advances the same trace.
- Reply intent creates one new trace and exact reply edge.
- Relay retry sends identical signed bytes and cannot mint a timestamp sibling.

## Attribution and Color Tests

- Initial MODEL turn renders entirely with its configured MODEL voice.
- AUTHOR replacement within a MODEL turn changes only replaced/inserted characters to the active AUTHOR voice.
- Untouched MODEL characters retain their MODEL voice.
- Internal Zine copy/paste preserves exact source Step evidence and source voice/color.
- External paste creates EXTERNAL Runs with the selected external-paste voice.
- FILESYSTEM reconciliation creates FILESYSTEM Runs distinct from EXTERNAL paste.
- Inline quoted source text is automatically bracketed/cited and retains source voice/color without requiring Coin.
- Historical Step selection reconstructs the historical colors even after later edits.
- Invalid or incomplete Run partitions fail closed and never silently attribute everything to the signer.

## Secret Canary

Use a unique credential such as `ZINE_TEST_SECRET_<random>`.

The canary may appear only:

- as input to the explicit credential-storage capability; and
- in the fake provider's received Authorization header.

After successful, failed, streamed, stopped, and UNKNOWN attempts, scan:

- captured IPC after provisioning;
- `press.sqlite3`, WAL, and operation/output tables;
- exact signed events and content;
- outbox records;
- renderer/native logs;
- typed faults and diagnostics exports;
- packaged-test traces and failure reports.

The fake provider must return a failure body that echoes the credential. Assert `OperationFault` contains only stable safe fields and never the raw body/canary.

## Stream Journal Tests

- Chunks have monotonic attempt-local sequence numbers.
- One native writer transaction may contain ready chunks from multiple attempts without interleaving their sequences.
- Coalescing respects 250 ms or 64 KiB, whichever comes first.
- Lifecycle transitions fsync immediately.
- Terminal success atomically stores one complete bounded response, deletes temporary chunks, and records success.
- Crash-truncated chunks recover as incomplete/ineligible.
- Oversized response, journal failure, disconnect, malformed SSE, and invalid UTF-8 all produce bounded typed faults.
- No test updates a cumulative response blob per delta.

## Replay and Performance Tests

- Cold-open a 1,000-turn cumulative task without decoding every historical snapshot.
- Switch among selected Steps and verify only the selected full body/Runs are decoded.
- Render/scroll virtualized transcript and timeline without retaining all historical Run arrays.
- Query a 10,000-event reply/context graph using indexed metadata.
- Measure peak memory, open latency, selected-Step latency, database/WAL growth, and compaction forecast.
- Exercise disk-full before AUTHOR commit, during stream journal, and during MODEL accept.
- Verify exact event blobs are stored once and outbox rows reference rather than copy them.

## Packaged macOS Journey

Use the embedded `@wdio/tauri-service` provider in a test-only build:

1. Launch clean installed app.
2. Configure fake GLM credential/provider.
3. Create task and AUTHOR draft.
4. Prepare exact context and inspect bytes.
5. Confirm `Step & Continue`; assert AUTHOR Step exists before provider response.
6. Stream fake response; reject once and assert no MODEL Step.
7. Run again; accept and assert one MODEL Step.
8. Kill/restart between commit and projection; verify identical transcript/colors/state/outbox.
9. Exercise ambiguous send and local Stop; verify UNKNOWN messaging.
10. Verify release build contains no WebDriver plugin/server capability.

## Real Relay Smoke

- Publish the exact AUTHOR and MODEL event bytes created from fixed fixtures.
- Fetch by event ID and compare the complete serialized event object.
- Retry identical bytes and verify relay deduplication/idempotency.
- Reject TaskOperation kind 4295 through ordinary publication.
- Test advertised message-size incompatibility and clear per-destination disclosure.

## Pass Criteria

- No provider I/O occurs before durable AUTHOR commit.
- Every confirmed AUTHOR turn exists exactly once after any later outcome.
- Only explicit Accept creates one MODEL Step.
- Exact inspected, persisted, and dispatched request bytes agree.
- Native signing cannot sign arbitrary webview templates.
- Secrets appear only at the two intentional canary boundaries.
- Restart reconstructs exact state after every named failpoint.
- Historical attribution and source colors remain exact.
- Test-only WebDriver capability is absent from production artifacts.
- No critical path is simultaneously untested, unhandled, and silent.
