## Affected Pages/Routes
- Desktop editor action palette — verify the existing single AI row still prepares, approves, and runs all five model operations plus Run.

## Key Interactions to Verify
- Inspect and approve a model operation, then change focus or content before completion and confirm the response is held rather than applied.
- Load valid and tampered Reify trace bundles and confirm signature, lineage, declaration, and materialized-file verdicts fail closed.
- Reload a local relay access policy through valid, corrupt, unreadable, deleted, and recreated states.
- Build the current-machine macOS dogfood app and DMG and inspect the embedded relay, ad-hoc signature, architecture, and unsafe-content report.

## Edge Cases
- Duplicate, dangling, malformed, oversized, and path-traversing Reify bundle fields.
- Broken provider headers, unsupported methods, UTF-8 error truncation, and split SSE frames in the native LLM proxy.
- Concurrent policy reads and writes, invalid operator requests, bans, hosted route traversal, and missing assets.
- MCP package execution with no network, no repository-relative imports, and no profile-state creation from `--help`.

## Critical Paths
- Shared protocol kernel consumers produce the same current Schema 1 trace behavior in client and MCP code.
- Approved AI bytes remain immutable from Inspector through provider transport and atomic application.
- `npm run verify` passes the protocol, client, MCP, relay, Rust, production-build, and real-relay checks on Node 24.
