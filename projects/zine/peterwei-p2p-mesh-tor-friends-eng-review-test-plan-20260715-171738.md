## Key Interactions to Verify
- Mint selected text and confirm a `mint/YYYY-MM-DD_HHmmss-smart-title.md` item appears.
- Open the minted item from the tree and from its resolved citation.
- Drag a minted item onto Root or an ordinary folder and confirm an editable fork appears while the Mint original remains.
- Confirm the source trace cites the minted node and the minted node records its exact `extracted-from` source.

## Edge Cases
- Two identical passages minted within the same second receive collision-free names.
- Mint stays flat and rejects ordinary moves into it.
- Mint and Oblivion content do not leak into ordinary Root presentation or authoring controls.
- A failed fork leaves the Mint original and tree state unchanged.
- Existing non-Mint moves and Oblivion restore behavior remain unchanged.

## Critical Paths
- Select passage -> Mint -> source citation -> open minted trace -> drag to Root -> edit fork.
- Reload after minting and confirm the Mint item and extraction provenance survive local persistence and relay stepping.

## Implementation Result
- Mint is a separate local-first protocol folder mounted beside Root, so sending Root cannot disclose Mint membership.
- Minted items use `YYYY-MM-DD_HHmmss-smart-title.md`, open from their citation, and remain immutable while still supporting explicit Send and post-Send Attest.
- Dragging a Mint item into Root or an ordinary Root folder creates an editable `forked-from` trace and preserves the original coin.
