## Affected Pages/Routes
- Desktop/web workspace editor citation bar and inbound footer: verify live citation state across moves, archive, restore, offline refresh, local deletion, and relay revocation.
- Workspace tree and trace context menus: verify identity-preserving move/rename/archive/restore plus separate local-delete and relay-revoke actions.

## Key Interactions to Verify
- A cites B, then A moves or renames: B still shows A under cited-by and the row opens A at its current location.
- A cites B, then A enters Oblivion: B hides A immediately, after relay convergence, and after restart.
- A is restored to any destination: the same trace identity and one current head are retained, and B shows A again.
- B enters Oblivion while active A cites it: A's outgoing chip remains and opens B read-only; B still shows active A in cited-by.
- Citation add, remove, picker, and drag/drop controls cannot mutate Mint, Oblivion, replay-frozen, or otherwise read-only traces.
- Deleting the local Oblivion copy does not revoke relay events; the separate Revoke action publishes NIP-09 and reports partial ownership honestly.

## Edge Cases
- Cached A1 inbound row while A2 becomes the local Oblivion/tombstone head.
- Current-head lookup timeout with a last verified result and with no prior verified result.
- Stale relay plus current relay return conflicting historical and current events.
- Legacy traces without a file TraceHead, malformed TraceHead content, multiple heads, broken prev links, and a prev cycle.
- Restore name collision, restore into a different folder, rapid archive/restore, restart between local move and relay publication, and retry after relay failure.
- Remote ordinary trace whose structural path begins with `oblivion/` is not treated as locally archived.
- Fork source archived/deleted; co-citation uses current active heads while Times retains historical event activity.
- Revocation signed by one owner on a chain containing foreign-signed nodes; only owned nodes and the owner's TraceHead address are requested for deletion.
- Cited event unavailable without a verifiable kind-5 request versus verifiably revoked by the cited event's signer.

## Critical Paths
- Real relay: cite -> cache -> edit -> archive -> restart -> restore elsewhere -> restart, asserting one trace identity, one selected head, correct inbound rows, and working navigation at every stage.
- Real relay: archive target B while active A cites it, then local-delete B and separately revoke B, asserting the outgoing chip transitions from archived-readable to unavailable/revoked without altering A's pinned node id.
- Client verification: focused lifecycle/provenance tests, full `npm test`, `npm run build`, and a real local relay exercise; run relay tests if NIP-09 handling requires relay changes.
