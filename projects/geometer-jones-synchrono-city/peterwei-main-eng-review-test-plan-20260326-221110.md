# Test Plan
Generated by /plan-eng-review on 2026-03-26
Branch: main
Repo: geometer-jones/synchrono-city

## Affected Pages/Routes
- `/app` — verify World map still renders, beacon markers pulse when live, marker selection opens the beacon panel, and the right panel behaves like beacon chat instead of the old place thread UI.
- `/app` — verify the active call bottom bar renders below the main content, shows media tiles for participants even when camera is off, and exposes mic/cam/screenshare/leave controls.
- `/app` on mobile widths — verify the beacon panel opens as the primary conversation surface without breaking the map-first World layout.
- `/app/chats` — regression check only: confirm private inbox behavior is unchanged in this phase and does not accidentally absorb beacon threads yet.

## Key Interactions to Verify
- Click a live beacon marker on the map, confirm the beacon card anchors from the avatar marker and the right panel opens the selected beacon conversation.
- Click `Join call`, confirm the active call state moves into the global bottom call bar and controls remain usable across navigation.
- Send multiple beacon chat messages in a row, confirm grouped-message rendering suppresses repeated avatar/name/timestamp except on the first item.
- Toggle mic, camera, and screenshare from the bottom call bar, confirm button state and participant media indicators stay in sync.
- Navigate between `World`, `Pulse`, and `Settings` while in a live call, confirm the bottom call bar persists and `Leave call` works from all routes.

## Edge Cases
- World bootstrap contains no beacons, verify empty state is still understandable and map shell does not break.
- Active call exists for a beacon not initially present in bootstrap, verify ephemeral beacon handling still works after the rename/projection layer.
- Beacon has text chat but no live participants, verify the panel renders without the live roster block.
- Live participants exist but camera is off, verify rectangular media tiles still show avatar/picture fallback rather than disappearing.
- Narrow/mobile viewport, verify the World screen remains map-first and the bottom call bar does not cover core controls.

## Critical Paths
- World bootstrap -> beacon projection -> map marker render -> beacon panel open
- Beacon panel -> `Join call` -> active call state -> bottom call bar -> leave call
- Beacon thread render -> grouped consecutive messages -> composer send path
- Existing private inbox route `/app/chats` remains DM/group-DM only in Phase 1
