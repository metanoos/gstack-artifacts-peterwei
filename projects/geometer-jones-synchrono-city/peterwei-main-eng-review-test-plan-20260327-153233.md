# Test Plan
Generated by /plan-eng-review on 2026-03-27
Branch: main
Repo: geometer-jones/synchrono-city

## Affected Pages/Routes
- `/app` — verify cohort beacon cards still open from the World map, show cohort-specific pinned-note context, and preserve explicit join semantics
- `/app?beacon=<geohash>` — verify cohort beacon detail shows current week / artifact context from pinned notes and supports listen-first call join
- `/app/settings` — verify room-permission admin flows still work and cohort host controls remain consistent with the fallback admin surface

## Key Interactions to Verify
- Open a cohort beacon from the map and inspect cohort metadata before joining
- Join a cohort room as a listener and verify the UI makes listen-only state obvious
- Promote a live listener to speaker and confirm controls unlock without forcing a rejoin
- Demote a live speaker back to listener and confirm publishing controls shut off cleanly
- Save or revoke room permissions for a cohort room and confirm host-facing UI stays in sync

## Edge Cases
- Cohort beacon missing a pinned note still renders safely with sensible fallback copy
- Non-cohort beacons do not inherit cohort-only join rules or host controls
- LiveKit participant update fails after permission save and the host sees a clear failure state
- Listener refreshes or rejoins mid-session and still lands in the correct publish/subscribe mode

## Critical Paths
- Newcomer inspects cohort beacon -> joins live as listener -> sees clear listen-only state
- Host promotes listener to speaker during a live session -> participant can publish without rejoining
- Host demotes speaker back to listener -> publishing controls stop and UI reflects the change
- Weekly pinned note remains the canonical cohort memory surface across World and Pulse reloads
