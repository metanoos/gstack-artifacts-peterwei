# Test Plan
Generated by /plan-eng-review on 2026-07-03
Branch: main
Repo: replay-editor (local, no remote)

## Affected Pages/Routes
- / (doc list + editor) — core writing loop; verify typing, Saved indicator honesty, doc create/rename/delete
- /r/<slug> (public replay) — published playback; verify logged-out playback identical to in-app, redacted content and images absent
- Editor replay view — timeline scrubbing, gap title-cards, speeds 1x/2x/4x/8x/16x/skim, multi-author attribution colors (post-Phase 4)

## Key Interactions to Verify
- Two devices editing the same doc alternately; kill one mid-sentence; both converge (post-Phase 1)
- Wifi flapping during typing: rapid disconnect/reconnect never duplicates or loses text
- AI proposal overlay: accept and reject inline; accept on two devices simultaneously → exactly one apply (post-Phase 2/3)
- IME (Chinese/emoji) composition while a remote edit arrives — one committed event, correct positions
- Time-travel restore: restore an earlier point, verify text matches and restore appears in replay
- Publish with a redacted span containing an image → image unfetchable on the public route

## Edge Cases
- Tab close mid-sentence (pagehide HTTP flush) — reopen shows every keystroke
- Second tab of same doc is read-only and live-updates
- Offline for hours on one device, then reconnect → converge without corruption
- Import a .docx/.md → first event is one origin:"paste" insert
- Corrupt persisted checkpoint → doc still opens (rebuild from log)

## Critical Paths
- Write → close → reopen → text intact (trust bar)
- Full co-writing loop: draft → invoke agent → accept/reject → replay shows dual attribution
- Publish → open logged out → identical playback → unpublish → 404
