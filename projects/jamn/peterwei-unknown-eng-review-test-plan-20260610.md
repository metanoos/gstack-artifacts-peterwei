## Affected Pages/Routes
- `/` (doc list) — create doc, open doc, docs persist across reload
- `/doc/:id` (editor) — typing records events; saved indicator only after flush; second tab opens read-only
- `/doc/:id/replay` (in-app replay) — playback with gap title-cards, speeds 1×/2×/4×/skim, backwards scrub
- `/r/:slug` (published replay, public) — logged-out playback identical to in-app preview; noindex header present; unguessable slug

## Key Interactions to Verify
- Type, close the tab mid-sentence, reopen — no characters lost
- Put laptop to sleep mid-sentence, resume — no characters lost
- Open the same doc in two tabs — second tab is read-only with a live view
- Go offline, keep typing, come back online — events sync; doc text identical on a second device
- Two devices edit the same doc while offline — on reconnect: clean rebase or an explicit fork; NEVER silently merged/corrupted text
- Paste a block of text — replay shows it as one instant beat, not synthetic typing
- Publish a doc — public URL plays the frozen snapshot; continuing to write does not change the public replay; re-publish replaces it
- Watch a replay across a session gap — "N days later" title card appears, honest duration label

## Edge Cases
- Empty document replay (zero events)
- Paste-only document (single large insert)
- Emoji / CJK / IME composition input — offsets stay consistent (UTF-16)
- Negative wall-clock deltas after a rebase — gap derivation clamps to zero (no backwards-running title cards)
- 100k-event document: skim completes in ≈60s; doc open does not hang
- Storage eviction simulation: unsynced-only events are the maximum possible loss

## Critical Paths
- Write → autosave → close → reopen → text + full replay intact (trust bar)
- Two-device offline divergence → rebase-or-fork, never corruption (merge policy)
- Publish → share link → logged-out viewer sees bit-identical playback (replay-core shared)
